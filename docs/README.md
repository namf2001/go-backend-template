# Documentation (`docs/`)

Thư mục này chứa toàn bộ tài liệu của dự án, hướng dẫn phát triển và quy trình làm việc.

## Cấu trúc

-   `phases/`: Hướng dẫn chi tiết từng bước (step-by-step guides) để xây dựng ứng dụng theo từng giai đoạn.
-   `api/`: Tài liệu API (OpenAPI/Swagger specs) nếu có.
-   `architecture/`: Biểu đồ kiến trúc, thiết kế hệ thống.

## Nội dung

Tài liệu nên được cập nhật liên tục khi code thay đổi để đảm bảo tính chính xác và hữu ích cho team member mới.
